# 1. Purpose

This document defines the **deployment strategy and infrastructure architecture** for the full system including:

* Frontend (Next.js)
* Backend (NestJS)
* AI Service (FastAPI RAG system)
* Database (PostgreSQL)
* Cache (Redis)
* File Storage (S3 / Cloudinary)
* Payment Gateway integrations

---

# 2. High-Level Deployment Architecture

```text
                USERS
                  ↓
          ┌──────────────┐
          │   FRONTEND   │
          │   Next.js    │
          │   (Vercel)   │
          └──────┬───────┘
                 │ API Calls
                 ↓
        ┌──────────────────┐
        │   BACKEND API    │
        │   NestJS Server  │
        │ (AWS / Railway)  │
        └──────┬───────────┘
               │
   ┌───────────┼──────────────┐
   ↓           ↓              ↓
DATABASE     REDIS        AI SERVICE
PostgreSQL   Cache       FastAPI RAG
(Cloud DB)   (Upstash)    (Python)
```

# 3. Frontend Deployment

## 3.1 Platform: Vercel (Recommended)

### Why Vercel:

* Zero-config deployment
* Automatic CI/CD
* Global CDN
* Built-in Next.js optimization

---

## 3.2 Deployment Flow

```text
GitHub Push → Vercel Build → CDN Deployment → Live Site
```

---

## 3.3 Environment Variables

```env
NEXT_PUBLIC_API_URL=https://api.yourdomain.com
NEXT_PUBLIC_PAYMENT_GATEWAY_KEY=xxxx
```

---

# 4. Backend Deployment

## 4.1 Recommended Options

### Option A (Recommended for Student Project)

* Railway
* Render

### Option B (Production Scale)

* AWS EC2
* AWS ECS (Docker)

---

## 4.2 Backend Deployment Flow

```text
GitHub → CI/CD Pipeline → Docker Build → Server Deploy → API Live
```

---

## 4.3 Backend Environment Variables

```env
DATABASE_URL=postgresql://...
JWT_SECRET=super_secret_key
REDIS_URL=redis://...
PAYHERE_MERCHANT_ID=xxxx
AI_SERVICE_URL=http://ai-service:8000
```

# 5. AI Service Deployment (RAG System)

## 5.1 Platform Options

* AWS EC2 (recommended)
* Render (simple deployment)
* Docker container service

---

## 5.2 AI Service Architecture

```text
NestJS Backend → FastAPI AI Service → ML Models → Response
```

---

## 5.3 AI Deployment Setup

* Python FastAPI server
* Uvicorn as ASGI server
* Docker container recommended

---

## 5.4 AI Environment Variables

```env
OPENAI_API_KEY=xxxx
DATABASE_URL=xxxx
MODEL_PATH=/models/
```

# 19. Conclusion

This deployment architecture ensures:

* Real-world production readiness
* Scalable cloud infrastructure
* Separation of services (frontend, backend, AI)
* Low-cost startup deployment
* Smooth upgrade path to enterprise system

> This system is designed to be deployable like a real SaaS product, not just a university project
