# 2. Core Innovation: Dual RAG Architecture

This system introduces **two separate AI assistants**:

---

## 2.1 Customer AI Assistant (Shopping Intelligence)

### Purpose:

Help customers find products faster without manual browsing.

---

### Example User Queries:

* “I need school uniform fabric for kids”
* “Show me cheap cotton shirts”
* “What is best fabric for hot weather?”

---

### AI Behavior:

The system:

1. Understands intent (NLP)
2. Searches product database (RAG retrieval)
3. Matches relevant products
4. Returns:

   * Product list
   * Direct product links
   * Price + stock info

---

### Response Format:

```json id="rag001"
{
  "query": "cotton shirt",
  "results": [
    {
      "productId": "uuid",
      "name": "Premium Cotton Shirt",
      "price": 2500,
      "stock": 20,
      "link": "/products/uuid"
    }
  ],
  "message": "These are the best matches for cotton shirts"
}
```

---

### Business Value:

* Faster product discovery
* Higher conversion rate
* Reduced user friction
* Increased sales

---

# 2.2 Owner AI Assistant (Business Intelligence RAG)

### Purpose:

Help business owner understand complex data using natural language.

---

### Example Queries:

* “What was the best-selling product last month?”
* “Which products are most profitable?”
* “What should I restock this week?”
* “Why did sales drop in June?”

---

### AI Behavior:

The system:

1. Retrieves structured business data
2. Combines with analytics engine
3. Applies RAG + LLM reasoning
4. Generates human-readable insights

---

### Response Example:

```json id="rag002"
{
  "query": "best selling last month",
  "insight": "Cotton School Uniforms were the top-selling product category last month with 32% of total sales.",
  "data": {
    "topProduct": "Cotton School Uniform",
    "revenue": 450000,
    "growth": "+18%"
  },
  "recommendation": "Increase stock by 25% for next month due to high demand trend."
}
```

---

### Business Value:

* Data-driven decision making
* Reduces dependency on manual reports
* Predictive business planning
* Improves profit margins

# 3. RAG System Architecture

## 3.1 High-Level Flow

```text id="ragflow001"
User Query
     ↓
Intent Detection Layer
     ↓
Route Decision:
   ├── Customer RAG
   └── Business RAG
     ↓
Retrieval Layer (Database / Vector Search)
     ↓
Context Enrichment
     ↓
LLM (GPT / OpenAI / Local Model)
     ↓
Final Response
```

---

# 3.2 RAG Components

## 1. Retriever Layer

Sources:

* PostgreSQL (Products, Orders)
* Sales Data
* Inventory Data
* Embedding Vector DB (future upgrade)

---

## 2. Embedding Layer (Future Upgrade)

* Convert product descriptions into vectors
* Enable semantic search
* Improve recommendation accuracy

Tools:

* OpenAI Embeddings OR
* Sentence Transformers

---

## 3. Generator Layer

* Large Language Model (LLM)
* Converts retrieved data into natural language

# 9. Security Design

* Admin AI restricted via role-based access
* Customer AI cannot access financial data
* API authentication required
* Query sanitization to prevent injection attacks

---

# 10. Performance Optimization

* Cache frequent queries
* Precompute analytics daily
* Limit LLM calls with smart routing
* Use batching for reports
